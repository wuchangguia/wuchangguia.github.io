    简单工厂模式

    1.代码实现

    -------------简单运算工厂类---------------
   public class OperationFactory{
    main(){
    Operation oper = null;
    switch (oper)
        {
            case "+":
                oper = new OperationAdd();        //OperationAdd()为具体的方法
                break;
            case "-":
                oper = new OperationSub();
                break;
            ......(略)
        }
    }
}
------------客户端代码------------------
    Operation oper;
    oper = OperationFactory.createOperate("+");
    oper.Num1 = 1;
    oper.Num2 = 2;
    double result = oper.GetResult();

    2.心得体会

    简单工厂模式分为三个结构：

    • 一为Factory类，为实现创建所有产品的内部逻辑，工厂类可以被外界直接调用，创建所用的对象。
    • 二为抽象产品类，为工厂类创建的所有产品的父类，封装了产品对象的公共方法，所有的具体产品为其子类对象
    • 三为具体产品类，即为工厂中创建的具体对象的实例，实现所有父类的抽象方法。

    结构图如下：

    简单工厂模式优点在于工厂类中进行逻辑判断，客户端的条件判断可以动态实例化对应的类，去除了客户端与具体产品的依赖

    简单工厂模式的缺点：不符合开放-封闭原则，例如上例中若要增加平方运算，需要修改原有的工厂运算类

    工厂方法模式

    1.代码实现

    -------------工厂接口----------------
interface IFactory
{
    Operation CreateOperation();
} 

-------------运算工厂类--------------- 
    class AddFactory implements IFactory{   
        public String createOperation() {
            return new OperationAdd();
        }
    }

    class SubFactory implements IFactory{
        public String createOperation() {
            return new OperationSub();
        }
    }
........(略)
------------客户端代码------------------

    IFactory operFactory = new AddFactory();
    Operation oper = operFactory.CreateOperation();
    oper.Num1 = 1;
    oper.Num2 = 2;
    double result = oper.GetResult();

    2.心得体会

    工厂方法模式的结构：

    • 一为抽象Factory类，为所有具体工厂的父类，所有工厂必须实现该抽象类中定义的方法。
    • 二为抽象产品类，为工厂类创建的所有产品的父类，封装了产品对象的公共方法，所有的具体产品为其子类对象
    • 三为具体产品类，即为工厂中创建的具体对象的实例，实现所有父类的抽象方法。
    • 四为具体Factory类，实现了抽象工厂的具体子类，返回一个具体产品的实例。

    结构图：

    工厂方法模式中子类决定实例化哪一个类，工厂方法使得每一个实例化延迟到子类

    总结

    • 工厂模式是基于面向对象语言中继承、多态特性而实现的。客户端调用的时父类接口，而具体实例化是在继承该接口的子类进行的，由工厂类决定逻辑即具体为哪一个子类。** 集中封装了对象的创建。**
    • 工厂模式实现时，客户端决定实例化哪一个工厂来实现具体的产品类，将逻辑判断给转移到了客户端的代码中进行。在简单工厂中，需要添加功能，修改工厂类，而在工厂方法模式中需要修改客户端的逻辑。利用反射可以解决分支判断的问题。
